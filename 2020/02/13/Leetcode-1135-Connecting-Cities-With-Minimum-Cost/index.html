1135. Connecting Cities With Minimum Cost
2020-02-13 | Leetcode | Leetcode Union Find

题目链接https://leetcode.com/problems/connecting-cities-with-minimum-cost/

题目难度Medium 56.2%

题目描述

There are N cities numbered from 1 to N.

You are given connections, where each connections[i] = [city1, city2, cost] represents the cost to connect city1 and city2 together. (A connection is bidirectional: connecting city1 and city2 is the same as connecting city2 and city1.)

Return the minimum cost so that for every pair of cities, there exists a path of connections (possibly of length 1) that connects those two cities together. The cost is the sum of the connection costs used. If the task is impossible, return -1.

Example 1:

Input: N = 3, connections = [[1,2,5],[1,3,6],[2,3,1]]
Output: 6
Explanation:
Choosing any 2 edges will connect all cities so we choose the minimum 2.

题目分析

等做完union find了再来写。

完整代码

class Solution {
    class UnionFind{
        public int count, total_cost;
        public int[] parent;

        public UnionFind(int n){
            count = n - 1;
            parent = new int[n];
            total_cost = 0;
            for(int i = 1; i < n; i++){
                parent[i] = i;
            }
        }

        public int find(int p){
            if(p == parent[p])
                return p;
            return parent[p] = find(parent[p]);
        }

        public void union(int p, int q, int cost){
            int rootp = find(p);
            int rootq = find(q);
            if(rootp != rootq){
                parent[rootq] = rootp;
                count--;
                total_cost += cost;
            }
        }
    }

    public int minimumCost(int N, int[][] connections) {
        Arrays.sort(connections, (n1, n2)->n1[2] - n2[2]); //先sort，然后往里加就应该是最小的cost
        UnionFind uf = new UnionFind(N + 1);
        for(int[] n : connections){
            uf.union(n[0], n[1], n[2]);
        }
        if(uf.count > 1) return -1;
        return uf.total_cost;
    }
}
